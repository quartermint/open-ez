# Plan 03-04: Defer CadQuery Imports in main.py

## Goal

Make `main.py` runnable without CadQuery installed, so that `--analysis`, `--validate`, `--summary`, `--validate-physics` modes work in any Python environment. Geometry modes (`--canard`, `--wing`, `--jigs`, `--generate-all`) will import CadQuery-dependent modules at call time and fail with a clear error if CadQuery is missing.

## Root Cause

`main.py` lines 21 and 23 eagerly import `core.structures` and `core.manufacturing` at module level, which triggers `import cadquery as cq` in 4 downstream files. This crashes before `argparse` even runs.

## Tasks

### Task 1: Move CadQuery-dependent imports into functions

**File:** `main.py`

**Changes:**

1. **Remove lines 21 and 23** (top-level imports of `core.structures` and `core.manufacturing`):
   ```python
   # REMOVE these two lines:
   from core.structures import CanardGenerator, WingGenerator  # noqa: E402
   from core.manufacturing import JigFactory  # noqa: E402
   ```

2. **Add local imports** in these 3 functions that use them:

   - `generate_manufacturing()` (line ~103): Add at top of function:
     ```python
     from core.structures import CanardGenerator
     from core.manufacturing import JigFactory
     ```

   - `generate_canard()` (line ~125): Add at top of function:
     ```python
     from core.structures import CanardGenerator
     ```

   - `generate_wing()` (line ~161): Add at top of function:
     ```python
     from core.structures import WingGenerator
     ```

3. **Keep line 22 unchanged** — `from core.analysis import physics, VSPBridge, OpenVSPRunner` does NOT depend on CadQuery.

**Verification:**
```bash
# Analysis mode works without CadQuery (system Python):
python3 main.py --analysis

# Validation works without CadQuery:
python3 main.py --validate

# Summary works:
python3 main.py --summary

# Tests still pass:
python3 -m pytest tests/ -x -q

# Native VSPAERO with python3.13:
python3.13 main.py --analysis
```

## Must-Haves

- [ ] `python3 main.py --analysis` completes without CadQuery
- [ ] `python3 main.py --validate` completes without CadQuery
- [ ] `python3 main.py --summary` completes without CadQuery
- [ ] `python3.13 main.py --analysis` prints "Using native VSPAERO"
- [ ] All existing tests pass (196+)
- [ ] Geometry modes still work when CadQuery IS available

## Risk

Minimal — this is the same pattern already used in main.py (see lines 43, 92, 127, 163, 231, 255 which all use local imports). No behavior change when CadQuery is installed.
