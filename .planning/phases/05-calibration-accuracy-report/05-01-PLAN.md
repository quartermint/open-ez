**Step 1: Investigate correct fs_wing_le value.**

The NP formula in PhysicsEngine.calculate_neutral_point() computes:
  ac_wing = fs_wing_le + y_mac * tan(sweep_le) + 0.25 * mac

The wing AC dominates the NP numerator (~80-85%). The 5.79" NP error maps approximately to a 5.79/0.82 = ~7.1" error in fs_wing_le. So the likely correct value is fs_wing_le = 133.0 - 7.1 = ~125.9 (internal), which corresponds to published FS ~80.4" (= 125.9 - 45.5).

To find the correct value:
1. Run the current PhysicsEngine.calculate_cg_envelope() and print NP, CG fwd, CG aft in published datum to confirm the pre-calibration baseline
2. Analytically derive the fs_wing_le correction: since NP_computed_published = 113.79 and NP_reference = 108.0, and the wing term dominates, compute the exact sensitivity dNP/d(fs_wing_le) from the existing code by examining calculate_neutral_point() in core/analysis.py
3. Use the sensitivity to compute the fs_wing_le value that yields NP within 2" of 108.0
4. If the analytical approach gives a clear answer (it should — the NP formula is linear in fs_wing_le), set fs_wing_le to that value
5. Source justification: "Calculated from NP delta analysis: published wing LE at root is approximately FS 80.4 (published datum), consistent with Long-EZ wing attachment point aft of cockpit. Pending CP-31 plan confirmation." This is per research recommendation.
