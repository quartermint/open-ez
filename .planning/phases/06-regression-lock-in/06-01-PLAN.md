Create precision regression tests locked to the externally-validated values from Phase 5's accuracy report.

Create `tests/test_regression_lock.py` with the following structure:

**Module docstring:** Explain these are Phase 6 regression tests locked to Phase 5 calibrated values. Each locked value traces to the accuracy report (which traces to reference_data.json / wind tunnel data). These tests catch future physics drift. DO NOT update locked values unless re-calibrating against external reference data.

**Setup:** Use the standard test pattern (REPO_ROOT, sys.path, CadQuery mock). Load reference_data.json and accuracy_report.json at module level for efficiency.

**Regression lock pattern for each PASS metric:**
Each test has TWO assertions:
1. **External truth check:** Computed value within reference_data.json tolerance of the published/measured reference value (catches gross errors)
2. **Drift detection check:** Computed value within a tight tolerance (0.01" for FS, 0.5 KTAS for speed, exact for config values) of the Phase 5 calibrated computed value (catches drift from code changes)

The drift detection tolerance is much tighter than the reference tolerance. This is the key innovation: reference tolerances catch physics errors, drift tolerances catch code regressions.

For stability metrics (NP, CG fwd, CG aft):
- Load reference value and tolerance from reference_data.json
- Run PhysicsEngine().calculate_cg_envelope()
- Convert to published datum via config.geometry.to_published_datum()
- Assert within reference tolerance (external truth)
- Assert within 0.01" of accuracy_report.json computed value (drift detection)

For stall speed:
- Use same first-principles calculation as in test_precision_validation.py and generate_accuracy_report.py
- Published reference areas from reference_data.json (94.2 + 15.6 sqft)
- Assert within 5.0 KTAS tolerance (external truth)
- Assert within 0.5 KTAS of accuracy_report computed value 53.2867 (drift detection)

For config values (gross weight, CLmax, alpha_0L):
- These are direct config reads -- assert exact match to reference_data.json value
- These also serve as regression locks ensuring config isn't accidentally modified

**Traceability test:**
- `test_regression_baselines_not_self_referential`: Read this test file's own source, assert "physics_baseline.json" does NOT appear as a data source (only as a string in the assertion itself is OK)
- `test_regression_values_match_accuracy_report`: Load accuracy_report.json, for each PASS metric verify the locked value in this test file matches the report's computed value

**Important:** Do NOT import or reference RegressionRunner or physics_baseline.json anywhere in this file. These tests are standalone, using PhysicsEngine and config directly. This is what breaks the self-referential loop.
