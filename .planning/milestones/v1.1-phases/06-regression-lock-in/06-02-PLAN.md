<objective>
Rewire `RegressionRunner` and `main.py` to read baselines from the Phase 5 accuracy report instead of the self-referential `physics_baseline.json`, and deprecate the old baseline file.

Purpose: This plan breaks the self-referential validation loop where `RegressionRunner` compared outputs against values it previously generated. After this change, the baseline source is `accuracy_report.json` which traces every metric to `reference_data.json` (published specs, wind tunnel data) or real VSPAERO runs. Future regressions are caught against external truth.

Output: Rewired `RegressionRunner`, updated `main.py`, updated `test_physics_regression.py`, deprecated `physics_baseline.json`.
</objective>

  <action>
Refactor `RegressionRunner` to use accuracy_report.json as its baseline source instead of physics_baseline.json.

**Changes to `load_baseline()`:**
Add a new method `load_accuracy_baseline(self, accuracy_report_path: Path) -> Dict[str, Dict[str, float]]` that:
1. Reads `data/validation/accuracy_report.json`
2. Extracts PASS metrics into the same `{scenario_name: {metric_name: value}}` format that `compare_to_baseline` expects
3. Maps accuracy report metric_ids to the existing scenario structure:
   - The 3 existing RegressionRunner scenarios (wing_reflex_moment, lift_curve_slope, spar_tip_deflection) were self-referential and used physics_baseline.json. The accuracy report has DIFFERENT metrics (NP, CG, stall speed, etc.).
   - The correct approach: change `compare_to_baseline` to accept the accuracy report format directly, comparing metric_id -> computed value pairs

**Key design decision:** Rather than contorting the accuracy report into the old scenario format, refactor `compare_to_baseline` to work with accuracy report format natively:

```python
def compare_to_accuracy_report(
    self,
    accuracy_report_path: Path,
    report_dir: Path,
) -> Tuple[bool, Dict[str, float], List[str]]:
    """Compare current physics outputs against accuracy_report.json baselines.

    Unlike the old compare_to_baseline (which compared against self-referential
    physics_baseline.json), this method compares against externally-validated
    values from the Phase 5 accuracy report. Each metric traces to
    reference_data.json (published specs, wind tunnel data).
    """
```

This method should:
1. Load accuracy_report.json
2. Filter to PASS metrics only (FAIL metrics are known gaps, not regressions)
3. Re-run the accuracy report metrics (import and call the relevant parts of `generate_accuracy_report.collect_metrics`)
4. Compare each current computed value against the accuracy report's computed value
5. Use each metric's tolerance_abs as the regression tolerance
6. Return (all_passed, current_values, failures) like the old method

**Keep the old methods** for backward compatibility but add deprecation comments:
```python
def load_baseline(self, baseline_path: Path) -> Dict[str, Dict[str, float]]:
    """DEPRECATED: Use compare_to_accuracy_report() instead.
    This method loads self-referential physics_baseline.json baselines.
    Retained for backward compatibility during Phase 6 transition."""
```

**Keep the old scenarios** but add deprecation comments explaining they produce self-referential values.

Add module-level constant:
```python
# Phase 6: Default baseline is now the accuracy report (externally-validated)
DEFAULT_ACCURACY_REPORT = Path(__file__).resolve().parents[2] / "data" / "validation" / "accuracy_report.json"
```
