<objective>
Fix the VSPCheckIsInit bug that silently disables the entire native VSPAERO integration path, then verify end-to-end with a real VSPAERO solver run.

Purpose: The `_try_import_vsp()` method calls `vsp.VSPCheckIsInit()` which does not exist in OpenVSP 3.48.2. This causes an `AttributeError` caught by the broad `except Exception` handler, setting `has_vsp=False` even though openvsp is installed and functional. As a result, `_run_native_sweep()` — despite being fully implemented — is never called. The pipeline always runs in surrogate mode.

Output: Fixed `_try_import_vsp()` using `vsp.GetVSPVersion()` as smoke test, real VSPAERO polar data in `vspaero_native_polars.json`, confirmed end-to-end native path.
</objective>
